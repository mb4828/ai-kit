# Development Standards

This skill enforces professional development standards for production-ready code in professional environments.

## When to Use This Skill

Apply this skill when:

- Writing code
- Editing code
- Reviewing code
- Code adjacent work (tests, documentation, etc.)

## Standards Documentation

Read the appropriate reference file(s) based on your task. Reading multiple files may be necessary if your work involves multiple aspects of development (e.g., both code and tests):

### For Writing or Reviewing Python Code

**Read `references/python-coding-standards.md`**

### For Writing or Reviewing Python Tests

**Read `references/python-testing-standards.md`**

### For Writing or Reviewing React + TypeScript Code

**Read `references/react-coding-standards.md`**

### For Writing or Reviewing React + TypeScript Tests

**Read `references/react-testing-standards.md`**

## Applying the Standards

1. **Identify your task** - Are you writing code, tests, or both?
2. **Read the relevant reference(s)** - Open the appropriate `.md` file(s)
3. **Apply the standards** - Follow the guidelines while writing
4. **Use the checklist** - Verify compliance before submitting
5. **Iterate if needed** - Refactor to meet standards if initial implementation doesn't comply

The goal is production-ready code that's maintainable, testable, and meets the standards expected in professional software development environments.
